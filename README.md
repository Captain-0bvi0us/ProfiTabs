# ProfiTabs

Офлайн PWA-приложение для создания и обмена гитарными/басовыми/барабанными табулатурами.

## Возможности

- Создание табов для гитары, электрогитары, бас-гитары и барабанов
- Редактирование с клавиатуры (десктоп) и тапом (мобильные)
- Воспроизведение (playback) с визуальным курсором
- Сохранение в локальное хранилище (IndexedDB)
- Экспорт табов в `.profitabs.json` файлы
- Импорт табов из файлов
- Обмен через «Поделиться» (Web Share API) / Telegram / почту / AirDrop
- Полная работа офлайн после установки как PWA

## Быстрый старт

```bash
cd frontend
npm install
npm run dev
```

Открыть на ноутбуке: `http://localhost:5173`

### Почему с телефона не открывается `localhost`

На телефоне `localhost` — это **сам телефон**, а не ваш ноутбук. С телефона нужно открывать **IP ноутбука в Wi‑Fi**, например `http://192.168.1.36:5173` (цифры смотрите в выводе `npm run dev` в строке `Network:`).

### Почему PWA «умирает», когда выключаешь ноутбук

Установленное приложение привязано к **адресу**, с которого вы его ставили (например `http://192.168.1.36:5173`). Пока ноутбук выключен, по этому адресу в сети **никто не отвечает**. Чтобы иконка открывалась **без ноутбука**, страница и скрипты должны уже лежать в **кэше service worker** — так надёжнее не dev, а **сборка + preview** (или статический хостинг).

Рекомендуемый порядок:

1. На ноутбуке в папке `frontend`: `npm run serve:lan` (это `build` + `vite preview` на `0.0.0.0`, порт **4173**).
2. В консоли появится строка `Network:` — открой этот URL **с телефона** в Chrome (та же Wi‑Fi сеть).
3. Подожди пару секунд (докачался service worker), при желании зайди в приложение ещё раз.
4. Установи PWA («Установить приложение» / «На главный экран»).
5. Проверка: включи на телефоне **режим полёта**, ноутбук можно выключить — если кэш подтянулся, приложение откроется из памяти телефона.

Если после выключения ноутбука пустой экран — один раз снова включи ноутбук, открой тот же `Network` URL с телефона, дождись полной загрузки, потом снова попробуй офлайн.

## Установка на телефон (PWA)

1. Предпочтительно: `npm run serve:lan`, с телефона открыть `http://<IP-ноутбука>:4173`
2. Либо только разработка: `npm run dev` и `http://<IP>:5173` (офлайн после выключения ПК менее предсказуем)
3. В Chrome на телефоне: «Установить приложение» / «Добавить на главный экран»
4. Данные табов хранятся на телефоне (IndexedDB), без вашего сервера

## Обмен табами

### Отправить:
1. Открыть таб → нажать **📤 Поделиться**
2. На Android: системное меню «Поделиться» (Telegram, почта, Bluetooth и т.д.)
3. На десктопе: скачивается `.profitabs.json` файл

### Получить:
1. Открыть **Мои табы** → нажать **📥 Импорт**
2. Выбрать полученный `.profitabs.json` файл
3. Таб появится в списке

## Деплой на GitHub Pages (рекомендуется для «одного URL на всегда»)

Сайт будет доступен как **`https://<ваш-логин>.github.io/<имя-репозитория>/`** (например `https://octocat.github.io/ProfiTabs/`).

### Что уже сделано в проекте

- В `vite.config.js` задан `base` через переменную **`VITE_BASE_PATH`** (при сборке в GitHub Actions подставляется **имя репозитория** автоматически).
- В `main.jsx` у роутера указан **`basename`**, чтобы маршруты (`/my-tabs`, `/editor` и т.д.) работали из подкаталога.
- В `public/.nojekyll` — чтобы GitHub Pages не обрабатывал сайт через Jekyll и не ломал пути.
- После сборки в CI копируется **`404.html`** = `index.html`, чтобы при обновлении страницы на вложенном URL открывалось SPA.
- Файл **`.github/workflows/deploy-pages.yml`** — сборка и публикация при пуше в `main` / `master`.

### Что сделать тебе

1. **Создай репозиторий на GitHub** (имя может быть любым, например `ProfiTabs`). Залей туда этот проект (корень репозитория = папка с `frontend` и `.github`).
2. **Settings → Pages → Build and deployment → Source:** выбери **GitHub Actions** (не «Deploy from a branch»).
3. **Запушь** в ветку `main` или `master`. Во вкладке **Actions** дождись зелёного workflow **Deploy to GitHub Pages**.
4. Через 1–2 минуты открой в браузере:  
   `https://<твой-логин>.github.io/<имя-репозитория>/`  
   Имя в URL **совпадает с именем репозитория** (регистр в URL GitHub обычно не важен, но путь в `VITE_BASE_PATH` берётся из имени репо как в GitHub).
5. **Установи PWA уже с этого адреса** — тогда приложение не привязано к ноутбуку и открывается из любой сети.

Если репозиторий переименуешь — первый деплой подставит новый путь; переустанови PWA с нового URL при необходимости.

### Локальная проверка «как на GitHub»

Подставь **то же имя**, что у репозитория (со слэшем в конце):

```powershell
cd frontend
$env:VITE_BASE_PATH="/ProfiTabs/"; npm run build; npm run preview
```

Открой `http://localhost:4173/ProfiTabs/` (Vite preview покажет точный URL в консоли).

### Деплой вручную

```bash
cd frontend
npm run build
# при необходимости: export VITE_BASE_PATH=/ИмяРепо/
```

Содержимое `dist/` можно выложить куда угодно; для GitHub Pages удобнее Actions выше.

## Технологии

- React 18 + Vite
- IndexedDB (через `idb`) для хранения данных
- Vite PWA Plugin (Workbox) для офлайн-работы
- Web Share API для обмена файлами
