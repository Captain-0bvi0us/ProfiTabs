# ProfiTabs

Приложение для создания и обмена гитарными табулатурами. PWA — работает офлайн после установки.

## Возможности

- **Редактор табулатур** для гитары, электрогитары, бас-гитары и барабанов
- Гитарные табы: цифры на 6 (или 4 для баса) струнных линейках
- Барабанные табы: сетка ударов по инструментам (kick, snare, hi-hat, toms, crash, ride)
- **Техники**: hammer-on, pull-off, bend, slide, vibrato, mute, palm mute, tapping, harmonics
- **Поиск пользователей** по имени и обмен табами
- **Офлайн-режим**: табы сохраняются в IndexedDB и доступны без интернета
- **PWA**: устанавливается на телефон через браузер

## Технологии

- **Backend**: Django + Django REST Framework + JWT-аутентификация
- **Frontend**: React + Vite + vite-plugin-pwa
- **БД**: SQLite (для локальной группы достаточно, легко мигрировать на PostgreSQL)
- **Офлайн**: Service Worker (Workbox) + IndexedDB (idb)

## Быстрый старт

### 1. Backend (Django)

```bash
cd backend

# Создать виртуальное окружение
python -m venv venv

# Активировать (Windows)
venv\Scripts\activate
# Активировать (Linux/Mac)
# source venv/bin/activate

# Установить зависимости
pip install -r requirements.txt

# Создать базу данных
python manage.py migrate

# Создать суперпользователя (для админки)
python manage.py createsuperuser

# Запустить сервер
python manage.py runserver
```

Backend будет доступен на `http://127.0.0.1:8000`. Админка: `/admin/`.

### 2. Frontend (React)

```bash
cd frontend

# Установить зависимости
npm install

# Запустить dev-сервер
npm run dev
```

Frontend будет доступен на `http://localhost:5173`. Vite автоматически проксирует `/api` запросы на Django.

### 3. Установка как PWA

1. Откройте `http://localhost:5173` на телефоне (оба устройства в одной сети)
2. В браузере Chrome → меню → «Добавить на главный экран»
3. Приложение установится и будет работать как нативное

## Как пользоваться

### Регистрация и вход
1. Откройте приложение, нажмите «Зарегистрироваться»
2. Введите имя пользователя, пароль и (по желанию) отображаемое имя
3. После регистрации вы автоматически войдёте в систему

### Создание табулатуры
1. На главной нажмите **«Новый таб»** или кнопку **+** в навигации
2. Заполните название, исполнителя, выберите инструмент, укажите BPM
3. **Для гитары/баса**: кликните на ячейку в сетке и введите номер лада (0-24) с клавиатуры. Стрелки для навигации. Delete — удалить ноту
4. **Для барабанов**: кликните на ячейку чтобы поставить/убрать удар. Пробел или X тоже переключает
5. Используйте кнопки техник (h, p, b, /, ~, x и т.д.) — выбранная техника применится к следующей введённой ноте
6. Нажмите **«+ Добавить такт»** чтобы расширить табулатуру
7. Нажмите **«Сохранить»**

### Обмен табами
1. Откройте таб → нажмите **«Поделиться»**
2. Найдите пользователя по имени → нажмите **«Отправить»**
3. Или перейдите в **«Люди»**, найдите пользователя, выберите таб из списка, отправьте

### Просмотр полученных табов
1. Перейдите в **«Обмен»** → вкладка **«Полученные»**
2. Непрочитанные отмечены фиолетовой полоской
3. Нажмите **«Сохранить офлайн»** чтобы таб был доступен без интернета

### Офлайн-режим
- Все ваши табы автоматически кэшируются при просмотре
- Полученные табы сохраняйте кнопкой «Сохранить офлайн»
- При отсутствии сети приложение покажет все сохранённые табы

## Структура сетки табулатуры

Каждый такт разбит на **16 долей** (шестнадцатые ноты). Каждая 4-я доля выделена (четвертная нота). Для стандартного размера 4/4:

```
Доли:  1 . . . 2 . . . 3 . . . 4 . . .
       |       |       |       |
       четверть
```

## Структура БД

- **User** — стандартная модель Django
- **Profile** — расширение: display_name, bio, instrument
- **Tab** — табулатура: title, artist, instrument, tempo, time_signature, tuning (JSON), data (JSON с тактами)
- **TabShare** — пересылка: from_user → to_user, tab, message, is_read

## API Endpoints

| Метод | URL | Описание |
|-------|-----|----------|
| POST | /api/auth/register/ | Регистрация |
| POST | /api/auth/token/ | Получить JWT |
| POST | /api/auth/token/refresh/ | Обновить JWT |
| GET | /api/auth/me/ | Текущий профиль |
| GET | /api/auth/users/?search=... | Поиск пользователей |
| GET/POST | /api/tabs/ | Список/создание табов |
| GET/PUT/DELETE | /api/tabs/:id/ | Детали/редактирование/удаление |
| POST | /api/shares/ | Отправить таб |
| GET | /api/shares/received/ | Полученные табы |
| GET | /api/shares/sent/ | Отправленные табы |
| POST | /api/shares/:id/read/ | Отметить как прочитанное |

## Деплой на Render (бесплатно)

### Шаг 1: Создать репозиторий на GitHub

```bash
cd ProfiTabs
git init
git add .
git commit -m "Initial commit"
```

Создайте репозиторий на github.com, затем:

```bash
git remote add origin https://github.com/ВАШ_ЮЗЕР/profitabs.git
git branch -M main
git push -u origin main
```

### Шаг 2: Зарегистрироваться на Render

1. Откройте https://render.com и войдите через GitHub
2. Нажмите **New → Blueprint** (или "New → Web Service")

### Шаг 3: Деплой через Blueprint (автоматически)

1. Нажмите **New → Blueprint**
2. Подключите репозиторий `profitabs`
3. Render прочитает `render.yaml` и создаст:
   - **Web Service** (Django + React) — бесплатно
   - **PostgreSQL** база данных — бесплатно
4. Нажмите **Apply** — деплой займёт 3-5 минут

### Шаг 3 (альтернатива): Ручной деплой

Если Blueprint не подхватился:

1. **New → PostgreSQL** → создайте БД (free), скопируйте Internal Database URL
2. **New → Web Service** → подключите репозиторий, настройте:
   - **Build Command**: `bash build.sh`
   - **Start Command**: `cd backend && gunicorn profitabs.wsgi:application --bind 0.0.0.0:$PORT`
3. Добавьте Environment Variables:
   - `DATABASE_URL` = Internal Database URL из шага 1
   - `DJANGO_SECRET_KEY` = любая длинная случайная строка
   - `DJANGO_DEBUG` = `False`
   - `DJANGO_ALLOWED_HOSTS` = `.onrender.com`
   - `PYTHON_VERSION` = `3.11.9`
   - `NODE_VERSION` = `20`

### Шаг 4: Создать первого пользователя

После деплоя откройте **Shell** в Render Dashboard:

```bash
cd backend && python manage.py createsuperuser
```

Или просто зарегистрируйтесь через интерфейс приложения.

### Шаг 5: Установить PWA на телефон

1. Откройте `https://profitabs.onrender.com` в Chrome на телефоне
2. Меню → **Добавить на главный экран**
3. Готово — приложение работает как нативное, табы доступны офлайн

### Обновления

При каждом `git push` в main — Render автоматически пересоберёт и задеплоит.

### Примечания

- Бесплатный тариф Render усыпляет сервер через 15 мин бездействия. Первый запрос после сна ~30 сек. Для группы из нескольких человек это терпимо.
- PostgreSQL на бесплатном тарифе удаляется через 90 дней — нужно пересоздать или перейти на платный ($7/мес).
- Для VPS-деплоя используйте `bash build.sh` + `gunicorn` за Nginx.
